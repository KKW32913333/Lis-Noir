# Lis Noir - CARD BATTLE

ドラゴンと聖女のゴシックファンタジー世界観を舞台にした、ブラウザで動くカードバトルゲーム（PWA対応）。

> 現在の状態：**プロトタイプ（コア機能が一通り遊べる段階）**

---

## 1. 技術スタック

- HTML / CSS / JavaScript のみ（ビルドツール・フレームワーク不使用）
- ブラウザでそのまま動作し、GitHub Pagesで無料公開可能
- 保存データは `localStorage`（ブラウザ内保存）
- フォント：Google Fonts（Cinzel / Noto Serif JP）をCDN読み込み

## 2. ファイル構成

```
index.html            画面のHTML構造 + 全CSS（<style>タグ内）
cg-game.js             ゲームロジック全部（カードデータ・状態管理・バトル処理）
manifest.json           PWA設定（アプリ名・アイコン・起動時の向き等）
apple-touch-icon.png    iOSホーム画面用アイコン (180x180)
icon-192.png            PWAアイコン (192x192)
icon-512.png            PWAアイコン (512x512)
icon-1024.png           PWAアイコン (1024x1024・ストア掲載等用)
favicon-32.png          ブラウザタブ用アイコン (32x32)
favicon-64.png          ブラウザタブ用アイコン (64x64)
hero-bg.jpg             ホーム画面ヒーロー背景（Lis Noirアートワーク）
card-fire-dragon.png    ファイアドラゴンのカードイラスト
sw.js                   Service Worker（オフラインキャッシュ）
README.md               本ファイル
```

すべてルート直下にフラットに配置（フォルダ分けなし）。GitHub Pagesは大文字小文字を区別するため、アップロード時はファイル名を変更しないこと。

## 4. カードの追加・差し替え方法

`cg-game.js` の `CARD_DEFS` にカード情報が1箇所にまとまっています。

```js
fire_dragon: {
  name: 'ファイアドラゴン', element: 'fire', rarity: 'legend',
  cost: 5, atk: 6, hp: 10, skill: '攻撃時、敵全体に2ダメージ',
  image: null,        // ← ここにファイル名を入れると画像が反映される（例: 'card-fire-dragon.png'）
  emoji: '🐉'          // image が null の間だけ使われるプレースホルダー
}
```

画像素材を追加する場合の流れ：
1. 白背景・被写体中心のプロンプトでAI画像生成
2. 生成画像をアップロード → 背景除去・トリミング・サイズ統一して加工
3. `image` フィールドにファイル名を登録

## 4.5 ショップの仕組み

`cg-game.js` の `SHOP_PACKS` にパック定義がまとまっています。

```js
{ id: 'rare', name: 'レアパック', icon: '🎁', currency: 'gems', cost: 10,
  desc: 'レア以上が確定で出るパック', weights: { normal: 0, rare: 65, epic: 28, legend: 7 } }
```

- `currency`: 消費する通貨（`gold` または `gems`）
- `weights`: レアリティごとの抽選重み（0にするとそのレアリティは出ない）
- 購入すると、重みに従って対象レアリティからカードをランダム抽選 → そのカードの経験値+20（100で1レベルアップ）
- 全カードは最初から所持済みという設計のため、パックは「新規解放」ではなく「強化素材（経験値）を引く」仕組みにしています

## 4.6 ミッションの仕組み

`cg-game.js` の `MISSIONS` に定義がまとまっています。

```js
{ id: 'win3', title: '勝利を重ねる', desc: 'バトルに3回勝利する', target: 3,
  check: s => s.totalWins || 0, reward: { gold: 500 } }
```

- `check`: 現在の進捗を state から算出する関数（勝利数・パック開封数・強化回数・デッキ枚数など）
- `target` に到達すると「受け取る」ボタンが有効化され、タップで `reward` の分だけゴールド/ジェムを付与
- 受け取り状況は `state.missionsClaimed` に保存され、再度は受け取れない
- 現在は全て「累計〇〇回」の実績型ミッション。日次リセット型のデイリーミッションは未実装（ホーム画面のデイリー報酬バーは現状まだ装飾のみ）

## 4.7 スペル/装備/フィールドカードの仕組み

`CARD_DEFS` の各カードに `type`（`monster`/`spell`/`equipment`/`field`、省略時は`monster`）と `target`、`effect` を持たせています。

```js
spell_fireball:  { name: 'ファイアボール', type: 'spell', target: 'enemy',
  effect: { kind: 'damage', value: 4 }, cost: 2, ... }
equip_shield:    { name: 'ガーディアンシールド', type: 'equipment', target: 'friendly',
  effect: { atk: 0, hp: 4 }, cost: 2, ... }
field_inferno:   { name: 'インフェルノフィールド', type: 'field', target: 'none',
  effect: { boostElement: 'fire', atk: 1 }, cost: 2, ... }
```

**操作方法**
- スペル（`target: 'enemy'`）: 手札で選択 → 敵モンスターまたは敵本体をタップして発動
- スペル（`target: 'none'`）: 手札でタップした瞬間に即発動（回復・ドロー等、対象選択不要）
- 装備（`target: 'friendly'`）: 手札で選択 → 自分の場のモンスターをタップして装備。攻撃力/HPが戦闘計算に加算される
- フィールド（`target: 'none'`）: 手札でタップした瞬間に即発動。場に出ている間、対応属性のモンスターの攻撃力を**両陣営とも**継続的に強化。新しいフィールドカードを出すと上書きされる（同時に1枚まで）

**現在の制限（既知の制約）**
- 敵AIはスペル/装備/フィールドカードを使用しません（モンスターの配置・攻撃のみ）
- 装備の攻撃力ボーナスは戦闘計算には正しく反映されますが、フィールド上のミニカード表示には現状バッジ表示されません（HP増加分はHPバッジに反映されます）

## 4.8 バトル画面のレベルアップ内容

**① 演出強化**
- ターン開始時に「YOUR TURN」「ENEMY TURN」のバナーが画面中央を横切る演出
- 攻撃・スペルダメージ発生時に画面シェイク＋赤いフラッシュのインパクト演出

**② ポートレートHUD**
- バトルヘッダーに自分のミニポートレート（Lis Noirのアートワーク使用）を表示
- バトル開始時に自分 vs 敵のVS演出（ポートレート・名前を表示、約1.4秒後に自動で戦闘開始）

**③ 戦況の見える化**
- 攻撃者（自分の場のモンスター）を選択すると、攻撃可能な敵1体ずつに「与えるダメージ数」のプレビューバッジを表示
- 属性相性が有利なら緑、不利なら赤でバッジの色が変化
- ダメージ系スペルを選択した場合も同様に効果値をプレビュー表示

**④ ステージ構造**
- `STAGES`（cg-game.js）に5ステージを定義。ステージごとに敵HP・出現カードのレアリティ傾向（弱い敵はノーマル中心、強い敵はエピック/レジェンド中心）・報酬（ゴールド/ジェム/トロフィー）が変化
- 「バトル」タブをタップするとステージ選択画面が表示され、未クリアの次ステージのみ解放（それ以降は🔒表示）
- ステージをクリアすると次のステージが解放され、`state.stageProgress` に記録

## 4.9 カード進化の仕組み

- `EVOLVE_LEVEL_REQ`(5) 以上のレベルで進化が解放され、`EVOLVE_COST`(💰800) を消費して進化
- 進化すると `state.cards[id].evolved = true` が保存され、以後そのカードは常に `⚔+${EVOLVE_BONUS_ATK}(2) ❤+${EVOLVE_BONUS_HP}(3)` のボーナス付きで表示・戦闘に使用される
- 進化ボーナスは「自分が所持しているカード」にのみ適用され、敵AIが同じIDのカードを使っても影響しません（`newBattleUnit(id, isPlayerCard)` で判定）
- 進化済みカードはミニ表示で光る枠＋★バッジが付きます（デッキ編成／カード一覧／手札／自分の場のカード）

## 4.10 カードイラストの差し替え実績

- `fire_dragon`（ファイアドラゴン）に、ユーザー提供のドラゴンイラストを適用しました（`card-fire-dragon.png`）
- 元画像はカード枠・ステータス数値が焼き込まれた完成品カード画像だったため、中央のイラスト部分のみを切り出して使用しています（アプリ側で独自にコスト/ステータス/レアリティ枠を描画するため、二重表示を避けるため）
- 今後カードイラストを追加する場合、**背景が透過/単色でない「完成品カード画像」でも、イラスト部分だけ切り出せば同様に組み込み可能**です

## 4.11 Service Worker（オフライン対応）について

- `sw.js` が `index.html` / `cg-game.js` / `manifest.json` / 画像素材一式をキャッシュし、オフラインでも起動できるようにしています
- **重要：今後キャッシュ対象ファイル（HTML/JS/画像等）を修正するたびに、`sw.js` 内の `CACHE_NAME` のバージョン番号を上げてください**（例: `lisnoir-cache-v1` → `v2`）。上げないと、ユーザーの端末で古いキャッシュが表示され続け「直っていない」ように見える原因になります。これは今後Claude側でも毎回対応します
- 新しい画像素材を追加した場合は `CACHE_ASSETS` 配列にファイル名を追加してください
- キャッシュ戦略は「キャッシュ優先→無ければネットワーク取得→取得できたらキャッシュに追加」。オフライン時にキャッシュも無い場合は `index.html` にフォールバックします
