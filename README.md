# Lis Noir - CARD BATTLE

ドラゴンと聖女のゴシックファンタジー世界観を舞台にした、ブラウザで動くカードバトルゲーム（PWA対応）。

> 現在の状態：**プロトタイプ（コア機能が一通り遊べる段階）**

---

## 1. 技術スタック

- HTML / CSS / JavaScript のみ（ビルドツール・フレームワーク不使用）
- ブラウザでそのまま動作し、GitHub Pagesで無料公開可能
- 保存データは `localStorage`（ブラウザ内保存）
- フォント：Google Fonts（Cinzel / Noto Serif JP）をCDN読み込み

## 3.9 期間限定ガチャ（チケット制）

- ショップ画面の一番上に、金縁の専用パネルで表示される特別なガチャ枠（`EVENT_GACHA_PACKS`、cg-game.js）
- 新通貨「🎫チケット」（`state.tickets`）を消費して回す専用ガチャ。初回ユーザーに1枚のみ初期配布（**それ以外の入手経路は未実装** — ミッション報酬やイベント報酬などに組み込む場合は追加対応が必要です）
- **「夜天の英雄ガチャ」**：以下6体の**固定プールから均等な確率で必ず1体**が出現（天井システムの対象外）。全て闇属性・レジェンド
  - ヴォイドリーパー／ノクターナルドラゴン／ルナエルフ／ナイトメアキャバリア／シャドウスライム／オービタルグリモア
- **通常パックとの排他性**：これら6体は、ノーマル/レア/プレミアムパックのレジェンド枠には出現しないよう`pickWeightedCardId()`で除外しています。専用ガチャでしか"新規に"引けない特別な存在として扱われます（ただし全カード自動所持の仕様上、実際には最初から使用可能。詳細は次項）
- 今後、期間限定ガチャを追加する場合は`EVENT_GACHA_PACKS`に1件追加するだけでOK

## 3.10 【重要】ガチャで引いたカードの所持・使用可否について

- **このゲームは「全カードを最初から所持している」仕様です。** `state.cards`は起動時に`CARD_DEFS`の全カードIDに対して自動生成され（Lv.1、進化前）、デッキ編成・カード一覧に**最初から表示され、使用できます**
- ガチャ・パックで同じカードを引くと、そのカードの**強化用経験値（+20exp）が加算される**仕組みです（新規解放ではなく、育成用の重複入手という位置づけ）
- そのため、今回追加した「夜天の英雄ガチャ」限定の6体も、**ガチャを引く前から一覧・デッキで使用可能**です。ガチャは「これらのカードを効率よく育成するための専用ルート」という扱いになります
- もし「ガチャを引くまでカードが未解放（ロック状態）」という、より一般的なソシャゲ的挙動にしたい場合は、`state.cards`の自動生成ロジックを変更し、未所持カードをカード一覧・デッキ編成画面で「？？？」表示にする対応が別途必要です。ご希望があれば実装します

## 4. カードの追加・差し替え方法

`cg-game.js` の `CARD_DEFS` にカード情報が1箇所にまとまっています。

```js
fire_dragon: {
  name: 'ファイアドラゴン', element: 'fire', rarity: 'legend',
  cost: 5, atk: 6, hp: 10, skill: '攻撃時、敵全体に2ダメージ',
  image: null,        // ← ここにファイル名を入れると画像が反映される（例: 'card-fire-dragon.png'）
  emoji: '🐉'          // image が null の間だけ使われるプレースホルダー
}
```

画像素材を追加する場合の流れ：
1. 白背景・被写体中心のプロンプトでAI画像生成
2. 生成画像をアップロード → 背景除去・トリミング・サイズ統一して加工
3. `image` フィールドにファイル名を登録

## 4.3 プレイヤー設定（名前・アイコン変更）

## 4.4 プレイヤーレベルの仕組み

- 従来は表示のみで実際にレベルアップしない不具合があったため、実際の経験値システムを実装
- `state.playerExp`（経験値）・`state.playerLevel`（レベル、初期値1）で管理
- **経験値の獲得**：バトルに勝利すると `10 + ステージ番号×6` の経験値を獲得（負けた場合は獲得なし）
- **必要経験値**：`20 + (レベル-1) × 12` で計算。序盤は少ない経験値ですぐレベルアップし、レベルが上がるほど必要量が増えていく仕様
- レベルアップすると、結果画面に「⭐ レベルアップ！ Lv.X」と表示される
- ホーム画面の「Lv.X」表示の下に、次のレベルまでの経験値バーを追加

## 4.5 ショップの仕組み

`cg-game.js` の `SHOP_PACKS` にパック定義がまとまっています。

```js
{ id: 'rare', name: 'レアパック', icon: '🎁', currency: 'gems', cost: 10,
  desc: 'レア以上が確定で出るパック', weights: { normal: 0, rare: 65, epic: 28, legend: 7 },
  preview: ['nature_treant', 'dark_wolf', 'light_angel'] }
```

- `currency`: 消費する通貨（`gold` または `gems`）
- `weights`: レアリティごとの抽選重み（0にするとそのレアリティは出ない）
- `preview`: パック一覧に表示する「収録例」のカードID（3枚程度。実際のイラストがあればそれを、無ければ絵文字プレースホルダーを表示）
- 購入すると、重みに従って対象レアリティからカードをランダム抽選 → **タップして開封するガチャ演出**（パックが震えながら光り、タップでバースト→カード獲得表示）を経て、そのカードの経験値+20（100で1レベルアップ）
- 全カードは最初から所持済みという設計のため、パックは「新規解放」ではなく「強化素材（経験値）を引く」仕組みにしています
- 通貨の消費・パック開封数のカウントは購入時点で即座に反映されるため、演出中に画面を離れても課金分は正しく処理されます
- **天井（保証）システム**：ノーマルが出る可能性のあるパック（ノーマルパック）で、ノーマルが10回連続で出ると、11回目は必ずレア以上が出るよう保証。パック一覧に「あと○回でレア以上確定」を表示。カウントは`state.pityCounters`にパックIDごとに保存

## 4.6 ミッションの仕組み

`cg-game.js` の `MISSIONS` に定義がまとまっています（2026-07-23時点で全25種）。

```js
{ id: 'win3', title: '勝利を重ねる', desc: 'バトルに3回勝利する', target: 3,
  check: s => s.totalWins || 0, reward: { gold: 500 } }
```

- `check`: 現在の進捗を state から算出する関数
- `target` に到達すると「受け取る」ボタンが有効化され、タップで `reward` の分だけゴールド/ジェムを付与
- **ミッションの種類**：勝利数（5段階）／パック開封数（4段階）／カード強化回数（3段階）／デッキ枚数（2段階）／トロフィー（2段階）／プレイヤーレベル（2段階）／ワールド制覇（5ワールド分）／カード進化数（2段階）
- **カテゴリ分類（2026-07-23追加）**：各ミッションに`category`（`battle`/`growth`/`collect`）を設定し、画面上部のフィルタタブ（全て／バトル／育成／収集）で絞り込み可能。増え続けるミッションを毎回スクロールして探す手間を軽減
  - バトル（10件）：勝利数・ワールド制覇
  - 育成（9件）：カード強化・トロフィー・プレイヤーレベル・カード進化
  - 収集（6件）：パック開封・デッキ編成
  - 各タブには、そのカテゴリ内の未受取ミッション件数が赤いバッジで表示される
- **まとめて受け取るボタン**：画面上部に、達成済みかつ未受取のミッションが1件以上あるときだけ表示される（`🎁 まとめて受け取る（n件）`）。タップで**フィルタに関わらず全カテゴリを対象に**一括受取
- 受け取り状況は `state.missionsClaimed` に保存され、再度は受け取れない
- 「累計〇〇回」の実績型ミッションに加え、日次リセット型の「デイリー報酬」も別途実装済み（4.14参照）

## 4.7 スペル/装備/フィールドカードの仕組み

`CARD_DEFS` の各カードに `type`（`monster`/`spell`/`equipment`/`field`、省略時は`monster`）と `target`、`effect` を持たせています。

```js
spell_fireball:  { name: 'ファイアボール', type: 'spell', target: 'enemy',
  effect: { kind: 'damage', value: 4 }, cost: 2, ... }
equip_shield:    { name: 'ガーディアンシールド', type: 'equipment', target: 'friendly',
  effect: { atk: 0, hp: 4 }, cost: 2, ... }
field_inferno:   { name: 'インフェルノフィールド', type: 'field', target: 'none',
  effect: { boostElement: 'fire', atk: 1 }, cost: 2, ... }
```

**効果の種類（`effect.kind`）**
- `damage`: 敵1体または敵本体にダメージ
- `heal`: 自分のHPを回復
- `draw`: 手札を追加で引く
- `wipe`: 相手の場のモンスターを**全て**撃破（`spell_apocalypse`「アポカリプス」で使用。`target: 'none'`のため手札タップで即発動）

**操作方法**
- スペル（`target: 'enemy'`）: 手札で選択 → 敵モンスターまたは敵本体をタップして発動
- スペル（`target: 'none'`）: 手札でタップした瞬間に即発動（回復・ドロー等、対象選択不要）
- 装備（`target: 'friendly'`）: 手札で選択 → 自分の場のモンスターをタップして装備。攻撃力/HPが戦闘計算に加算される
- フィールド（`target: 'none'`）: 手札でタップした瞬間に即発動。場に出ている間、対応属性のモンスターの攻撃力を**両陣営とも**継続的に強化。新しいフィールドカードを出すと上書きされる（同時に1枚まで）

**現在の制限（既知の制約）**
- 敵AIはスペル/装備/フィールドカードを使用します（4.8.2参照）が、行動判断は単純な優先順位ルールベースです（高度な戦略性はありません）
- 装備の攻撃力ボーナスは戦闘計算には正しく反映されますが、フィールド上のミニカード表示には現状バッジ表示されません（HP増加分はHPバッジに反映されます）

- 全25種のモンスターカードに `role`（`attacker` または `defender`）を設定。カード表示の右上に⚔（アタッカー）/🛡（ディフェンダー）のバッジで区別
- **属性ごとの内訳（2026-07-23 バランス調整済み）**：全属性にアタッカー・ディフェンダー両方が存在するよう再調整（全体で アタッカー13 : ディフェンダー12）

  | 属性 | アタッカー | ディフェンダー |
  |---|---|---|
  | 火 | 2 | 2 |
  | 水 | 3 | 3 |
  | 自然 | 3 | 3 |
  | 光 | 2 | 2 |
  | 闇 | 3 | 2 |

  - 調整前は「闇属性がアタッカー5・ディフェンダー0」「自然属性がアタッカー1・ディフェンダー5」という極端な偏りがあり、闇デッキ相手にはアタッカー/ディフェンダーのルールが実質機能しない状態でした。以下6種のロール・ステータスを見直して是正：
    - `fire_imp`（フレイムインプ）：アタッカー→ディフェンダー（atk2/hp1 → atk1/hp3）
    - `water_slime`（ブルースライム）：アタッカー→ディフェンダー（atk1/hp2 → atk1/hp3）
    - `nature_panda`（テンペストエレメンタル）：ディフェンダー→アタッカー（atk2/hp4 → atk4/hp3）

    - `dark_demonlord`（デモンロード）：アタッカー→ディフェンダー（atk6/hp7 → atk5/hp9）
- **ルール**：
  - アタッカーは、相手の場にディフェンダーがいる場合、いずれかのディフェンダーしか攻撃できない（いなければ他のモンスターや本体を自由に攻撃可能）

  - このルールは敵AIにも同様に適用される（`getValidTargets()`で共通判定）
- 攻撃可能なモンスターを選択すると、攻撃できない敵（ルール上ブロックされている）は暗く表示され、有効な対象にのみダメージプレビューが表示される

## 4.10.2 モンスタースキルの実装修正（重要）

**発見した問題**：`fire_dragon`（旧ファイアドラゴン）の全体攻撃スキルを除き、**モンスターの固有スキルはテキスト表示のみで、実際には一切発動していませんでした**（`撃破された時に復活する`、`場に出た時にHPを回復する`等、17種のスキルが全て未実装の飾りでした）。

**対応**：`skillTag`という構造化データをカード定義に追加し、以下のトリガー/効果を実際にバトルロジックへ実装しました。プレイヤー・敵AI双方に対称的に適用されます。

| トリガー | 効果 | 該当カード |
|---|---|---|
| `onPlay`（場に出た時） | `healSelf`（自分回復） | アクアゴーレム |
| `onPlay` | `healAllAllies`（味方全体回復） | ガーディアンエンジェル、エルフユニコーン |
| `onPlay` | `drawCard`（カードを引く） | クリスタルフォックス、シルフ |
| `onAttack`（攻撃時） | `extraDamage`（追加ダメージ） | サンダーイーグル |
| `onAttack` | `aoeDamage`（敵全体に追加ダメージ） | フレイムドレイク |
| `onAttack` | `stunTarget`（攻撃した相手を1ターン行動不能に） | リヴァイアサン |
| `onKillAttack`（撃破した時） | `drainEnemyCost`（相手のコストを減少） | ブレイズデーモン、デモンロード |
| `onDeath`（撃破された時） | `reviveHalfHp`（1度だけ半分のHPで復活） | サンフェニックス |
| `turnStart`（ターン開始時） | `healSelf`（自分回復） | ウッドエント、マグマコロッサス |

- スタン状態は場のカードに💫アイコンで表示

## 4.14 セーブデータについて

- **クラウド保存（推奨）**：ホーム画面右上の「⚙」→「設定・データ管理」から、メールアドレス＋パスワードで新規登録/ログインできます。ログインすると、状態が変わるたびに自動でFirestore（Firebase）に保存され、別端末やSafariキャッシュ削除後でも同じアカウントでログインすれば復元できます
- **未ログイン時**：`localStorage`（端末のブラウザ内）にのみ保存されます。アプリの再起動では残りますが、Safariの「履歴とWebサイトデータを消去」を実行すると消えます
- **手動バックアップコード**：クラウド保存を使わない場合の代替手段として、セーブデータをテキストコード化してコピー/貼り付けで復元できる機能も残しています
- ログイン時にクラウドにデータが見つかった場合、「クラウドを読み込む」か「今の端末のデータを優先する」かを選べます

### Firebase側の設定（開発者向け）
- 使用サービス：Firebase Authentication（メール/パスワード）、Cloud Firestore
- 設定は `firebase-init.js` に記述（`firebaseConfig` はプロジェクト「lis-noir」のもの。Web APIキーは公開情報扱いのためコードに直書きで問題なし。実際のアクセス制御はFirestoreセキュリティルールで行う）
- Firestoreのコレクション：
  - `saves`：ドキュメントID = ユーザーのUID、フィールド `data`（セーブデータ全体をJSON文字列化したもの）。本人のみ読み書き可能
  - `leaderboard`：ドキュメントID = ユーザーのUID、フィールド `displayName`・`trophy`・`updatedAt`。ランキング表示用に**全員が読み取り可能**（書き込みは本人のみ）
- **必須のセキュリティルール**（Firebaseコンソール → Firestore Database → ルール タブに設定。`leaderboard` の行を追加済み）：
```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /saves/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
    match /leaderboard/{userId} {
      allow read: if true;
      allow write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```
これにより、セーブデータ本体は本人のみ読み書き可能、ランキング用データは誰でも閲覧できるが本人しか更新できない状態になります。

## 4.11 Service Worker（オフライン対応）について

- `sw.js` が `index.html` / `cg-game.js` / `manifest.json` / 画像素材一式をキャッシュし、オフラインでも起動できるようにしています
- **重要：今後キャッシュ対象ファイル（HTML/JS/画像等）を修正するたびに、`sw.js` 内の `CACHE_NAME` のバージョン番号を上げてください**（例: `lisnoir-cache-v1` → `v2`）。上げないと、ユーザーの端末で古いキャッシュが表示され続け「直っていない」ように見える原因になります。これは今後Claude側でも毎回対応します
- 新しい画像素材を追加した場合は `CACHE_ASSETS` 配列にファイル名を追加してください
- キャッシュ戦略は「キャッシュ優先→無ければネットワーク取得→取得できたらキャッシュに追加」。オフライン時にキャッシュも無い場合は `index.html` にフォールバックします
